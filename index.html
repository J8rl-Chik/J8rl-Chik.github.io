TROUBLESHOOTING

#01 협업 환경에서 배포 권한 문제로 인한 팀원의 번거로움

프로젝트 초기, 개인 GitHub 저장소를 생성한 뒤 팀원을 협력자(Collaborator)로 초대해 개발을 진행했다. 나는 저장소 소유자였기 때문에 배포 과정에서 아무 문제를 겪지 않았지만, 협력자 권한만 가진 팀원은 Vercel 배포 과정에서 권한 문제에 막혔다. 팀원은 저장소 소유자가 아니라는 이유로 직접 배포가 어려웠고, 임시방편으로 저장소를 Fork해 자신의 저장소에서 배포를 진행했다. 하지만 Fork 저장소는 원본과 자동으로 동기화되지 않아, 변경 사항이 생길 때마다 Sync Fork를 수동으로 실행해야 하는 번거로움이 생겼다.
